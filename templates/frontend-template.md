## ⚙️ REQUIREMENTS

### 1. COMPONENT DESIGN

- Prefer small, composable components
- Avoid unnecessary abstraction
- Keep logic close to usage when possible

---

### 2. STATE MANAGEMENT

- Use local state when possible
- Introduce global state only if necessary
- Avoid over-engineering state structure

---

### 3. API INTEGRATION

- Handle loading, error and empty states
- Keep API logic separated from UI when possible
- Avoid tight coupling between UI and data layer

---

### 4. UX REQUIREMENTS

- Provide clear feedback to user actions
- Handle edge cases gracefully
- Avoid layout shifts and UI flickering

---

### 5. PERFORMANCE

- Avoid unnecessary re-renders
- Lazy load when appropriate
- Optimize rendering of lists and heavy components
